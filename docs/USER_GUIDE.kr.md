## 1. EML 이란 무엇인가?

### 핵심 아이디어

세상의 모든 초등 함수(지수함수, 로그함수, 삼각함수, 거듭제곱 등)가 사실은 단 하나의 이항 연산자로부터 만들어질 수 있다면 어떨까요?

Odrzywołek의 2026년 논문은 바로 그 연산자를 발견했습니다.

$$\operatorname{eml}(x,\, y) = e^x - \ln y$$

이 하나의 연산자와 두 가지 단말 노드 — 상수 **1** 과 변수 **x** — 만 있으면, 아래 단순한 문법이 복소수 평면 위의 모든 초등 함수를 이진 트리로 생성합니다.

```
S  →  1  |  x  |  eml(S, S)
```

### 몇 가지 놀라운 예시

| 함수            | EML 표현                                                                       |
| --------------- | ------------------------------------------------------------------------------ |
| $e^x$           | $\operatorname{eml}(x,\; 1)$                                                   |
| $e$ (오일러 수) | $\operatorname{eml}(1,\; 1)$                                                   |
| $\ln x$         | $\operatorname{eml}(1,\; \operatorname{eml}(\operatorname{eml}(1,\; x),\; 1))$ |
| $-x$            | 위 세 가지를 조합한 깊이 8의 트리                                              |
| $\sqrt{x}$      | 깊이 33의 트리                                                                 |

$e^x$는 "exp 방향으로 x를 내려보내고, ln 방향으로 상수 1을 내려보낸다"는 단 한 단계의 연산입니다.

### 왜 흥미로운가?

- **아날로그 컴퓨팅**: 하드웨어에 단 하나의 회로 소자만 있으면 모든 함수 계산이 가능합니다.
- **기호 회귀(Symbolic Regression)**: 데이터에서 수식을 자동 발견하는 탐색 공간을 극도로 단순화합니다.
- **수학적 미**: 복잡한 세계가 단순한 규칙 하나로 설명됩니다.

EML Tree Visualizer는 이 변환 과정을 눈으로 직접 탐색할 수 있는 도구입니다.

## 2. 화면 구성

```
┌──────────────────────────────────────────────────────────┐
│  [수식 입력창]                     [Standard | Optimized] │  ← 입력 영역
│  ─────────────────────────────────────────────────────── │
│  Standard  [exp(x)] [ln(x)] [e] [exp(exp(x))] [ln(ln(x))]│  ← 예시 버튼
│  Optimized [exp(ln(x))] [ln(exp(x))]                     │
│  Deep(K≫19) sin(x)  cos(x)                               │
├──────────────────────────────────────────────────────────┤
│                                                          │
│            EML 트리 캔버스 (React Flow)                   │  ← 메인 뷰
│                                                          │
│  [−] [+] [fit]                              [minimap]    │
├──────────────────────────────────────────────────────────┤
│  RPN: x1E    K=3  3 nodes  depth 1  ✓ optimal  [Copy]    │  ← 결과 바
├──────────────────────────────────────────────────────────┤
│  ▼ x =  1.00  [────●────────────]  f(x) = 2.718282+0i    │  ← 파라미터 패널
└──────────────────────────────────────────────────────────┘
```

| 영역                | 역할                                            |
| ------------------- | ----------------------------------------------- |
| **입력 영역**       | 수식 입력, 모드 전환, 예시 선택                 |
| **EML 트리 캔버스** | 변환된 이진 트리를 인터랙티브하게 표시          |
| **결과 바**         | RPN 문자열, 복잡도(K), 노드 수, 깊이, 최적 여부 |
| **파라미터 패널**   | x 값 설정 후 f(x) 계산 결과 확인                |

## 3. 수식 입력하기

### 기본 입력

상단 입력창에 수식을 타이핑하면 **350ms** 후 자동으로 변환됩니다. `Enter` 키를 누르면 즉시 변환됩니다.

```
exp(x)         → 지수함수
ln(x)          → 자연로그
sqrt(x)        → 제곱근
x^2            → 제곱
-x             → 부호 반전
x + 1          → 덧셈
x * x          → 곱셈
x / 2          → 나눗셈
exp(x) + ln(x) → 합성 수식
```

### 지원 상수

| 입력               | 의미                               |
| ------------------ | ---------------------------------- |
| `1`                | EML의 기본 단말 노드               |
| `e`                | 오일러 수 ≈ 2.71828                |
| `2`, `3`, ... `10` | 정수 상수 (1을 반복 덧셈으로 구성) |
| `0`                | 영 (zero)                          |

> **주의:** `pi`, `i`, `11`, `3.14` 같은 상수는 아직 지원하지 않습니다. $\pi$는 K=193 이상의 트리가 필요하며, 탐색 범위를 벗어납니다.

### 오류가 발생했을 때

입력창 아래에 빨간 오류 메시지가 나타납니다. 예시:

```
Function 'sin' is not directly supported in Standard Mode.
→ "Try Optimized →" 버튼이 나타나면 클릭하여 Optimized Mode로 전환
```

## 4. 컴파일 모드 이해하기

오른쪽 상단의 토글 버튼으로 두 모드를 전환합니다.

### Standard Mode (표준 모드)

**특징:** 즉각적인 결과, 수백 밀리초 이내

논문의 심볼릭 컴파일 규칙을 그대로 적용합니다. 결과 트리는 수학적으로 정확하지만, **K-최솟값(최소 복잡도)을 보장하지는 않습니다.**

지원하는 모든 함수에 대해 즉시 트리를 생성합니다. `sqrt(x)`, `-x`, `x+1` 같은 복잡한 수식도 처리하며, 트리가 수백 노드가 되더라도 정확한 값을 계산합니다.

```
exp(x)  → K=3   (최적)
ln(x)   → K=7   (최적)
sqrt(x) → K=139 (유효하나 최솟값 아님)
-x      → K=17  (유효하나 최솟값 아님; 논문의 최솟값은 K≈57)
```

### Optimized Mode (최적화 모드)

**특징:** 더 오래 걸리지만 K-최솟값 탐색

K=1부터 K=13까지, 가능한 모든 RPN 문자열을 열거하며 수치적으로 일치하는 최단 트리를 찾습니다. Web Worker 백그라운드에서 실행되므로 **UI가 멈추지 않습니다.**

탐색 중에는 진행 상황이 표시됩니다:

```
Searching… K=5 · 14,532 candidates
```

탐색 후 Standard Mode의 결과보다 짧은 트리가 발견되면 즉시 교체합니다.

```
exp(ln(x)) → Standard K=9  →  Optimized K=1 (= x, 단순 항등함수!)
ln(exp(x)) → Standard K=9  →  Optimized K=1 (= x)
```

탐색이 K=13 범위 내에서 끝나면 아래 메시지가 나타납니다:

```
Not found within K≤13 (20,134 candidates searched).
This function likely requires a much deeper EML tree.
```

> **팁:** `exp(ln(x))`나 `ln(exp(x))`를 Optimized Mode에서 시도해 보세요. 수학적으로 단순화되어 K=1의 단일 노드 `x`가 됩니다!

## 5. 트리 시각화 읽는 법

### 노드 종류

```
┌─────────────┐
│ eml         │   ← 내부 노드: 직사각형, "eml" 표시
│  exp │  ln  │      왼쪽 = exp(좌측 자식), 오른쪽 = ln(우측 자식)
└──────┴──────┘
   │         │
  (좌)       (우)
 exp방향    ln방향

 ┌───┐      ┌───┐
 │ 1 │      │ x │   ← 단말 노드: 원형
 └───┘      └───┘
상수(검정)  변수(파랑)
```

### 엣지 라벨

각 엣지에는 `exp` 또는 `ln` 라벨이 붙습니다. 이 라벨이 **비가환성** 을 시각적으로 알려줍니다.

- **왼쪽 엣지 (exp):** 이 자식 노드의 값이 `exp(·)` 안으로 들어갑니다
- **오른쪽 엣지 (ln):** 이 자식 노드의 값이 `ln(·)` 안으로 들어갑니다

$\operatorname{eml}(x, y) = e^x - \ln y$ 이므로, 왼쪽과 오른쪽이 **반드시 다른 역할**을 합니다.

### 캔버스 조작

| 동작        | 방법                                 |
| ----------- | ------------------------------------ |
| 이동        | 빈 공간을 클릭 후 드래그             |
| 줌 인/아웃  | 마우스 휠 또는 좌하단 `+` / `-` 버튼 |
| 전체 맞추기 | 좌하단 `fit` 버튼                    |
| 미니맵 확인 | 우하단의 축소 지도로 전체 트리 파악  |

### 트리가 너무 클 때

`sqrt(x)` 나 `-x` 등 심볼릭 컴파일 결과는 노드가 수십~수백 개입니다. 이 경우:

1. **미니맵**으로 전체 구조를 파악하세요.
2. **`fit` 버튼**을 눌러 화면에 맞게 줄여서 보세요.
3. 마우스 휠로 확대하면 특정 부분을 자세히 볼 수 있습니다.

## 6. RPN 표기법과 K 복잡도

### RPN(역폴란드 표기법)이란?

EML 트리를 후위 순회하여 한 줄 문자열로 직렬화한 표현입니다.

| 토큰 | 의미            |
| ---- | --------------- |
| `1`  | 상수 단말 노드  |
| `x`  | 변수 단말 노드  |
| `E`  | eml 연산자 노드 |

**읽는 방법:** RPN 문자열을 왼쪽부터 읽으면서 스택에 쌓고, `E`가 나오면 상단 두 개를 꺼내 `eml(아래, 위)`를 만듭니다.

**예시: `11xE1EE` = ln(x)**

```
읽기 순서: 1 → 1 → x → E → 1 → E → E

스택 변화:
[1]
[1, 1]
[1, 1, x]
[1, eml(1,x)]        ← E: pop x, pop 1 → eml(1, x)
[1, eml(1,x), 1]
[1, eml(eml(1,x), 1)] ← E: pop 1, pop eml(1,x) → eml(eml(1,x), 1)
[eml(1, eml(eml(1,x), 1))] ← E: 최종 결과

= exp(1) - ln(eml(eml(1,x), 1))
= e - ln(exp(e - ln(x)))
= e - (e - ln(x)) = ln(x) ✓
```

### K 복잡도

**K = RPN 문자열의 길이** = 트리의 전체 노드(단말 + 연산자) 수

K가 작을수록 더 "간결한" EML 표현입니다. 이것이 논문에서 말하는 **Kolmogorov 복잡도**의 EML 버전입니다.

| K           | 총 후보 수 | 탐색 시간 |
| ----------- | ---------- | --------- |
| 1           | 2          | < 1ms     |
| 3           | 4          | < 1ms     |
| 5           | 14         | < 1ms     |
| 7           | 42         | < 1ms     |
| 9           | 132        | < 1ms     |
| 11          | 429        | < 10ms    |
| 13          | 1,430      | < 100ms   |
| 합계 (K≤13) | 20,134     | < 1s      |

### 결과 바에서 확인할 수 있는 정보

```
RPN: 11xE1EE    K=7   7 nodes   depth 3   [Copy]
                 ▲      ▲         ▲
            RPN 길이  트리 노드수  최대 깊이
```

`✓ optimal` 표시가 있으면 Optimized Mode에서 K-최솟값 트리가 확인된 결과입니다.

## 7. 파라미터 패널로 값 계산하기

화면 하단의 파라미터 패널을 펼치면(▼ 클릭) x 값을 변경하고 함수값을 확인할 수 있습니다.

### 슬라이더 사용

```
x =  2.00  [───────●────────────]  →  f(x) = 7.389056 + 0.000000i
              -10            +10
```

- 슬라이더를 드래그하거나 숫자 입력창에 직접 값을 입력합니다.
- 범위: -10.0 ~ +10.0

### 결과 읽기

결과는 항상 복소수 형태로 표시됩니다.

```
f(x) = 7.389056 + 0.000000i   → 실수 결과 (exp(2) ≈ 7.389)
f(x) = 0.000000 + 1.000000i   → 허수 결과
f(x) = Infinity                → 발산 (예: ln(0))
f(x) = NaN                    → 정의 불가
```

### 활용 팁

- `ln(x)` 입력 후 x = 1 → f(x) = 0 확인
- `exp(ln(x))` Optimized Mode → K=1 (단순 항등함수, f(x) = x)
- `sqrt(x)` 입력 후 x = 4 → f(x) = 2.000 확인
- `-x` 입력 후 x = 3 → f(x) = -3.000 확인

## 8. 예시 모음 활용법

입력 영역 아래에 세 줄의 예시 버튼이 있습니다.

### Standard 예시

| 버튼          | 수식          | 설명                          |
| ------------- | ------------- | ----------------------------- |
| `1`           | `1`           | 가장 단순한 단말 노드, K=1    |
| `e`           | `e`           | 오일러 수, K=3                |
| `exp(x)`      | `exp(x)`      | 지수함수, K=3 (최적)          |
| `ln(x)`       | `ln(x)`       | 자연로그, K=7 (최적)          |
| `exp(exp(x))` | `exp(exp(x))` | 이중 지수, K=5                |
| `ln(ln(x))`   | `ln(ln(x))`   | 이중 로그, K=13               |
| `sqrt(x)`     | `sqrt(x)`     | 제곱근, K=139 (심볼릭 컴파일) |
| `x^2`         | `x^2`         | 제곱, K=75                    |
| `-x`          | `-x`          | 부호 반전, K=17               |

### Optimized 예시

| 버튼         | 수식         | 설명                             |
| ------------ | ------------ | -------------------------------- |
| `exp(ln(x))` | `exp(ln(x))` | Standard K=9 → **Optimized K=1** |
| `ln(exp(x))` | `ln(exp(x))` | Standard K=9 → **Optimized K=1** |

> **주목!** `exp(ln(x)) = x` 이므로, Optimized Mode는 이를 단 하나의 노드 `x`(K=1)로 발견합니다. 수학적 항등식을 자동으로 "발견"하는 셈입니다.

### Deep (K ≫ 19) — 정보 표시용

`sin(x)`, `cos(x)` 는 복소수 중간값이 필요하고 K가 수백을 넘어 현재 탐색 범위(K=13)를 크게 벗어납니다. 클릭할 수 없는 점선 테두리로 표시됩니다.

## 9. 실전 예시 따라하기

### 예시 1: ln(x) 트리 이해하기

1. 입력창에 `ln(x)` 입력
2. 결과 바에서 `RPN: 11xE1EE  K=7` 확인
3. 트리를 읽어보기:
   - 루트: `eml(1, 하위트리)`
   - 하위트리: `eml(eml(1, x), 1)`
   - 수식으로: $\exp(1) - \ln(\exp(\exp(1) - \ln(x))) = e - (e - \ln x) = \ln x$ ✓
4. 파라미터 패널에서 x = e (≈ 2.718) 입력 → f(x) = 1.000 확인

### 예시 2: Optimized Mode로 항등식 발견하기

1. 오른쪽 상단 토글을 `Optimized`로 전환
2. 입력창에 `exp(ln(x))` 입력
3. 탐색 시작 메시지 확인: `Starting search…`
4. 수초 후 K=1 결과 도착: `RPN: x  K=1  1 nodes  depth 0  ✓ optimal`
5. 캔버스에 원형 노드 `x` 하나만 표시됨
6. 해석: $e^{\ln x} = x$ — 수학적으로 자명하지만, 알고리즘이 자동으로 이를 발견!

### 예시 3: sqrt(x) 심볼릭 트리 탐색

1. 입력창에 `sqrt(x)` 입력 (Standard Mode)
2. 결과 바: `K=139  33 depth` — 매우 큰 트리
3. `fit` 버튼을 눌러 전체 트리를 화면에 맞춤
4. 미니맵으로 전체 구조 파악
5. 파라미터 패널에서 x = 4 → f(x) = 2.000 확인 ← 수학적으로 정확!

### 예시 4: 복합 수식 입력

1. 입력창에 `x^2 + 1` 입력
2. Standard Mode에서 즉시 트리 생성 (K ≈ 100)
3. x = 3 → f(x) = 10.000 (3² + 1 = 10) 확인
4. Optimized Mode로 전환 — K=13 범위 내에 더 짧은 형식이 없음을 확인:
   ```
   Not found within K≤13 (20,134 candidates searched).
   ```

## 11. 자주 묻는 질문

**Q: Standard Mode 결과의 K가 크게 나왔습니다. 더 짧은 트리가 있지 않나요?**

A: 있을 수 있습니다! Optimized Mode를 사용하면 K=13 이내에서 더 짧은 표현을 탐색합니다. 단, K=13 이상이 필요한 경우 Optimized Mode도 찾지 못할 수 있습니다.

---

**Q: `exp(ln(x))`를 Optimized Mode로 돌리면 K=1이 나오는데, 이게 맞나요?**

A: 수학적으로 완전히 맞습니다. $e^{\ln x} = x$ 이므로, 이 함수는 항등함수이고 단일 노드 `x` (K=1)로 충분합니다. Optimized Mode가 이 항등식을 자동 발견한 것입니다.

---

**Q: `sin(x)`를 입력하면 오류가 나옵니다.**

A: `sin(x)`는 EML 트리로 표현하려면 복소수 중간값과 오일러 공식이 필요하며, K가 수십~수백에 달합니다. 현재 구현에서는 아직 지원하지 않습니다. 향후 더 깊은 탐색 알고리즘(Adam optimizer 기반)이 추가될 예정입니다.

---

**Q: 트리가 화면 밖으로 나갑니다.**

A: 좌하단의 `fit` 버튼을 클릭하면 트리 전체를 화면에 맞게 자동 조정합니다. 미니맵(우하단)을 참고해 현재 위치를 파악할 수도 있습니다.

---

**Q: Standard Mode 결과가 잘못된 값을 계산하나요?**

A: 아닙니다. Standard Mode의 트리는 K가 최솟값이 아닐 수 있지만, **수학적으로 항상 정확합니다.** 파라미터 패널에서 직접 확인해 보세요. `sqrt(4) = 2`, `-3 = -3` 등 정확한 결과를 제공합니다.

---

**Q: RPN을 직접 입력해서 트리를 그릴 수 있나요?**

A: 현재는 지원하지 않습니다. 수식을 텍스트로 입력하면 자동 변환됩니다. 직접 RPN을 입력하는 기능은 향후 추가 예정입니다.

---

**Q: Optimized Mode 탐색이 오래 걸립니다.**

A: K=13까지 약 20,134개의 후보를 탐색합니다. 보통 1초 이내에 완료됩니다. 탐색 중 다른 수식을 입력하면 즉시 기존 탐색을 중단하고 새로운 탐색을 시작합니다.

## 부록: 핵심 수식 증명

### exp(x) = eml(x, 1)

$$\operatorname{eml}(x, 1) = e^x - \ln 1 = e^x - 0 = e^x \checkmark$$

### ln(x) = eml(1, eml(eml(1, x), 1))

$$\operatorname{eml}(1, x) = e^1 - \ln x = e - \ln x$$
$$\operatorname{eml}(e - \ln x,\ 1) = e^{e - \ln x} - 0 = \frac{e^e}{x}$$

잠깐, 이것은 우변이 $\frac{e^e}{x}$입니다. 다시 올바르게 전개합니다:

$$\operatorname{eml}(\operatorname{eml}(1,x),\ 1) = \exp(\operatorname{eml}(1,x)) - \ln(1) = \exp(e - \ln x)$$
$$\operatorname{eml}(1,\ \exp(e - \ln x)) = e - \ln(\exp(e - \ln x)) = e - (e - \ln x) = \ln x \checkmark$$

### e = eml(1, 1)

$$\operatorname{eml}(1, 1) = e^1 - \ln 1 = e - 0 = e \checkmark$$
